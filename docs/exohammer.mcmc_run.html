explore_iteratively(total_iterations, checkpoints, burnin_factor=0.2, thinning_factor=0.001, moves=<emcee.moves.de.DEMove object>, nwalkers=None, verbose=True, tune=True, silent=False)[source]

The primary method for running an MCMC sampler.

Parameters:

  • total_iterations (int) – The total number of steps to advance your chains

  • checkpoints (int) – At this number of steps, the run will save your run for incremental evaluation

  • burnin_factor (float) – A normalized percentage of the run to discard as burn in. Note that the entire chain is saved in memory and burnin_factor adjusts the burn in value at each checkpoint.

  • thinning_factor (float) – A normalized percentage of the run to thin the entire run by. Useful for production-length runs where the chains must take >10^7 steps. Must be at most 1% of burnin_factor

  • moves (object) – The type of moves for the Ensemble Sampler to generate. See https://emcee.readthedocs.io/en/stable/tutorials/moves/ for more information

  • nwalkers (int) – If you wish to specify the number of walkers in your ensemble, define this value Otherwise, the default will be the closest even value of 3 times the number dimensions.

  • verbose (bool) – Whether to display a generic tqdm progress bar for your run.

  • tune (bool) – Passed directly into emcee.EnsembleSampler.sample. It is highly recommended that this value remain True

  • silent (bool) – Whether to display your plots in your IDE or not. Useful for testing, but this should be set to False during production runs.
